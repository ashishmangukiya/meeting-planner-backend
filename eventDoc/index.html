Events to listen

verify both User and admin event
This event ("verify") has to be listened on both the user's end and admin's end to verify user authentication and admin authentication.
user and admin both will only be set as online after verification of authentication token.
online-user-list event
This event ("online-user-list") has to be listen on admin's end to get updated online user list .
and the online-user-list will be in the form of object(userId:userName);
get-errors event
This event ("get-errors")has to listen on both the end so if any error occured on socket then error will be notified to user or admin
userId event
This event ("userId") has to be listened on user side . whenever admin is creating any meeting or updating meeting or deleting any meeting schedule at the same time user will get notification for meeting schedule and meeting schedule will be added to calendar page
adminId event
This event ("adminId") has to be listened on admin side. admin will get notification whenever user stops reminder.

Events to emit

set-user event
This event ("set-user") has to be emitted when user or admin get verify .
if user/admin authorized using authToken then user/admin will be added to online-list.
user-message event
This event ("user-message") has to be emitted when meeting schedule is created/updated by admin.
meeting schedule will be sent to selected user through this event.
delete-message event
This event ("delete-message") has to be emitted when any meeting schedule has been deleted by admin.
and on server side meeting schedule will be deleted from database.
get-reminder event
This event ("get-reminder") has to be emitted when admin send reminder meeting schedule to user and on user side user will get notification for reminder.
send-message-offline event
This event ("send-message-offline") has to be emitted when admin send reminder for meeting schedule but user should be offline.
if user is offline and user has reminder meeting schedule then this event will be emitted.
stop-reminder event
This event ("stop-reminder") has to be emitted when user stops reminder notification.
once user stops reminder notification user will not get notification for stopped reminder meeting schedule.
disconnection
This event ("disconnection") has to be emitted, when the user gets disconnected.
